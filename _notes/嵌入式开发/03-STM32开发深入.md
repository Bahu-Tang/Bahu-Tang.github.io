# 03 - STM32 开发深入

> 从"能让灯亮"到"能掌控芯片"，掌握 STM32 核心外设。

---

## STM32 家族速览

| 系列 | 核心 | 主频 | 价格 | 适合 |
|------|------|------|------|------|
| F0 | Cortex-M0 | 48MHz | ~5元 | 简单控制 |
| **F1** | **Cortex-M3** | **72MHz** | **~10元** | **学习首选** |
| F4 | Cortex-M4 | 168MHz | ~25元 | 带 FPU+DSP |
| H7 | Cortex-M7 | 480MHz | ~60元 | 高性能 |

**我们的主力**：STM32F103C8T6（Blue Pill），性价比最高。

---

## 开发环境双修方案

### STM32CubeIDE

**适用场景**：想图形化配置外设、快速原型

**核心工作流**：
1. 打开 CubeMX（.ioc 文件）
2. 图形化配置引脚（设置 GPIO/定时器/串口 模式）
3. 配置时钟树（HSE → PLL → 72MHz）
4. Generate Code → 生成 HAL 初始化代码
5. 在 `/* USER CODE BEGIN */` 之间写业务逻辑
6. Build → Debug/Run

### PlatformIO + VSCode

**适用场景**：习惯命令行/轻量环境、代码量大时更流畅

**核心工作流**：
1. `platformio init --board bluepill_f103c8`
2. 手动配置 `platformio.ini`
3. 安装所需库（`lib_deps`）
4. 写代码 → Build → Upload

**platformio.ini 示例**：
```ini
[env:bluepill_f103c8]
platform = ststm32
board = bluepill_f103c8
framework = stm32cube
upload_protocol = stlink
monitor_speed = 115200
```

### 混合方案（推荐）

1. 用 STM32CubeMX 生成初始化代码（.ioc 图形化配置）
2. 导入到 PlatformIO 或 STM32CubeIDE 继续开发
3. 写逻辑代码时在 VSCode / PlatformIO 更舒服

## 核心外设速查

### 1. GPIO（通用输入输出）

**最基础**，控制 LED、读按键。

```c
// 初始化（CubeMX 生成）
// 或手动代码：
__HAL_RCC_GPIOA_CLK_ENABLE();
GPIO_InitTypeDef gpio = {0};
gpio.Pin = GPIO_PIN_0;
gpio.Mode = GPIO_MODE_OUTPUT_PP;  // 推挽输出
gpio.Pull = GPIO_NOPULL;
gpio.Speed = GPIO_SPEED_FREQ_LOW;
HAL_GPIO_Init(GPIOA, &gpio);

// 使用
HAL_GPIO_WritePin(GPIOA, GPIO_PIN_0, GPIO_PIN_SET);
HAL_GPIO_WritePin(GPIOA, GPIO_PIN_0, GPIO_PIN_RESET);
HAL_GPIO_TogglePin(GPIOA, GPIO_PIN_0);  // 翻转
uint8_t state = HAL_GPIO_ReadPin(GPIOB, GPIO_PIN_1); // 读取
```

### 2. 定时器 + PWM（脉宽调制）

**调亮度、控制舵机**。核心：改变占空比。

```c
// 初始化定时器 PWM（CubeMX 配置 TIM2_CH1 → PA0）
// 手动启动：
HAL_TIM_PWM_Start(&htim2, TIM_CHANNEL_1);

// 设置占空比（0~999，对应 0%~100% 如果 ARR=999）
__HAL_TIM_SET_COMPARE(&htim2, TIM_CHANNEL_1, 500); // 50% 亮度
```

**LED 调光**：ARR=999, PSC=71 → 频率 = 72MHz/(72*1000) = 1kHz → 改变 CCR 值即可调光。

**舵机控制**：ARR=19999, PSC=71 → 频率 50Hz（20ms 周期）→ CCR=500~2500 对应 0°~180°。

### 3. 外部中断

**按按钮响应**，不占用 CPU 轮询。

```c
// CubeMX → GPIO → EXTI 模式
// 中断回调：
void HAL_GPIO_EXTI_Callback(uint16_t GPIO_Pin) {
    if (GPIO_Pin == GPIO_PIN_3) {
        // 按键按下，切换灯色
        current_mode = (current_mode + 1) % 3;
    }
}
```

### 4. 串口 UART

**和电脑/ESP32 通信**。红绿灯升级：串口发指令控制灯色。

```c
// CubeMX 配置 USART1：PA9=TX, PA10=RX, 115200-8-N-1
uint8_t rx_buf[1];

// 中断接收一个字节
HAL_UART_Receive_IT(&huart1, rx_buf, 1);

// 接收回调
void HAL_UART_RxCpltCallback(UART_HandleTypeDef *huart) {
    if (huart == &huart1) {
        switch(rx_buf[0]) {
            case 'R': set_light(0); break;
            case 'Y': set_light(1); break;
            case 'G': set_light(2); break;
        }
        HAL_UART_Receive_IT(&huart1, rx_buf, 1); // 继续接收
    }
}

// 发送
HAL_UART_Transmit(&huart1, (uint8_t*)"Hello\r\n", 7, 100);
```

### 5. I2C（两线串行总线）

**连传感器**：MPU6050 陀螺仪、OLED 屏幕等。

```c
// CubeMX 配置 I2C1：PB6=SCL, PB7=SDA
// 读写寄存器：
HAL_I2C_Mem_Read(&hi2c1, DEV_ADDR<<1, REG_ADDR, 1, &data, 1, 100);
HAL_I2C_Mem_Write(&hi2c1, DEV_ADDR<<1, REG_ADDR, 1, &data, 1, 100);
```

### 6. ADC（模数转换）

**读模拟传感器**：光敏电阻、电位器。

```c
// CubeMX 配置 ADC1_IN0 (PA0)
HAL_ADC_Start(&hadc1);
if (HAL_ADC_PollForConversion(&hadc1, 100) == HAL_OK) {
    uint32_t value = HAL_ADC_GetValue(&hadc1); // 0~4095 (12位)
}
```

### 7. DMA（直接存储器访问）

**不占 CPU 搬数据**，配合串口/ADC 使用。大量数据传输场景才需要，初学可选学。

## 时钟树速查

STM32F103 最高 72MHz：

```
外部晶振 8MHz (HSE)
→ PLL 倍频 ×9 = 72MHz (SYSCLK)
→ AHB 分频 /1 = 72MHz (HCLK)
→ APB1 分频 /2 = 36MHz (PCLK1, 给 TIM2~7)
→ APB2 分频 /1 = 72MHz (PCLK2, 给 TIM1, GPIO, ADC)
```

在 CubeMX 的 Clock Configuration 页面图形化配置即可。

## 调试技巧

### 1. 串口打印（最常用）

```c
// 重定向 printf 到串口
int __io_putchar(int ch) {
    HAL_UART_Transmit(&huart1, (uint8_t*)&ch, 1, 100);
    return ch;
}
// 然后就能用 printf("value=%d\r\n", x); 了
```

### 2. LED 心跳

让一个 LED 在 while(1) 里以 1Hz 闪烁，确认程序没卡死。

### 3. HardFault 排查

如果程序跑飞了，检查：
- 数组越界
- 使用未初始化的外设
- 中断里干了太多事
