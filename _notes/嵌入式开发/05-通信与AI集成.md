# 05 - 通信与 AI 集成

> 让单片机联网，调用 AI API，成为真正的"AI Agent 物理化身"。

---

## 整体架构

```
┌──────────┐    UART     ┌──────────┐    WiFi     ┌────────────┐
│  STM32   │◄──────────►│ ESP32/01 │◄──────────►│  AI API     │
│ (控制核) │            │ (通信核)  │            │ (OpenAI等)  │
└──────────┘            └──────────┘            └────────────┘
     │                                            ┌──────────┐
     ├─ LED灯/舵机/电机                            │ OpenCode  │
     ├─ 传感器读取                                  │  Agent    │
     └─ 本地决策                                    └──────────┘
```

- **STM32**：负责硬实时控制（舵机、电机、LED、传感器采集）
- **ESP32**：负责网络通信和 AI 交互（WiFi、HTTP/MQTT、语音处理）
- 两者通过 UART 串口通信

---

## 通信路线选择

### 路线 A：ESP8266 (ESP-01S)

- 便宜（~8元），体积小
- 只有 WiFi，通过 AT 指令集通信
- 适合简单场景：只做 HTTP 请求

### 路线 B：ESP32（推荐）

- ~20 元，WiFi + 蓝牙 + 双核 MCU
- 可以直接当主控用，也可以仅当 WiFi 协处理器
- GPIO 多，可单独驱动传感器和舵机
- 有 Arduino 框架，代码好写

**建议**：直接上 ESP32，省心。

## STM32 ↔ ESP32 串口通信协议

### 简易协议设计

```
帧格式：$CMD,PARAM1,PARAM2\r\n
响应：   $OK,PARAM\r\n 或 $ERR,MSG\r\n
```

示例：
```
发送:  $LED,GREEN\r\n
响应:  $OK\r\n

发送:  $AGENT,STATUS\r\n
响应:  $OK,IDLE\r\n

发送:  $SPEAK,你好我是机器人\r\n
响应:  $OK\r\n
```

### STM32 端代码框架

```c
// 串口接收缓冲区
#define RX_BUF_SIZE 128
char rx_buf[RX_BUF_SIZE];
uint8_t rx_idx = 0;
uint8_t rx_complete = 0;

// 串口中断接收
void HAL_UART_RxCpltCallback(UART_HandleTypeDef *huart) {
    // 逐字节接收
    char c = rx_byte;
    if (c == '\n') {
        rx_buf[rx_idx] = '\0';
        rx_complete = 1;
        rx_idx = 0;
    } else if (rx_idx < RX_BUF_SIZE - 1) {
        rx_buf[rx_idx++] = c;
    }
    HAL_UART_Receive_IT(&huart1, &rx_byte, 1);
}

// 主循环处理
void process_cmd(char *cmd) {
    if (strncmp(cmd, "$LED,GREEN", 10) == 0) {
        set_light(2);
        uart_send("$OK\r\n");
    } else if (strncmp(cmd, "$LED,RED", 8) == 0) {
        set_light(0);
        uart_send("$OK\r\n");
    }
}
```

## ESP32 端：联网与 AI 调用

### 环境

用 PlatformIO 或 Arduino IDE，`framework = arduino`。

### WiFi 连接

```cpp
#include <WiFi.h>

const char* ssid = "你的WiFi名";
const char* password = "你的密码";

void setup() {
    WiFi.begin(ssid, password);
    while (WiFi.status() != WL_CONNECTED) {
        delay(500);
    }
    // 已连接
}
```

### HTTP 请求（调用 AI API）

```cpp
#include <HTTPClient.h>

String call_ai_api(String prompt) {
    HTTPClient http;
    http.begin("https://api.openai.com/v1/chat/completions");
    http.addHeader("Content-Type", "application/json");
    http.addHeader("Authorization", "Bearer YOUR_API_KEY");
    
    String body = "{\"model\":\"gpt-4o-mini\",\"messages\":[{\"role\":\"user\",\"content\":\"" + prompt + "\"}]}";
    
    int code = http.POST(body);
    String response = http.getString();
    http.end();
    
    return response;
}
```

### MQTT 方案（更适合实时通信）

```cpp
#include <PubSubClient.h>

WiFiClient espClient;
PubSubClient mqtt(espClient);

void setup() {
    mqtt.setServer("broker.emqx.io", 1883);  // 免费公共 MQTT Broker
    mqtt.setCallback(mqtt_callback);
}

void mqtt_callback(char* topic, byte* payload, unsigned int length) {
    // 收到消息，通过串口发给 STM32
    Serial2.print("$MQTT,");
    Serial2.write(payload, length);
    Serial2.print("\r\n");
}

void loop() {
    mqtt.loop();
    // STM32 发来的指令
    if (Serial2.available()) {
        String cmd = Serial2.readStringUntil('\n');
        if (cmd.startsWith("$MQTT_PUB")) {
            mqtt.publish("robot/status", "online");
        }
    }
}
```

## 红绿灯终极版：WiFi 联网自动查询 Agent 状态

### 方案设计

```
ESP32 定时(每5秒) → HTTP GET /agent/status → 解析JSON
                                                │
                                   ┌────────────┼────────────┐
                                   ▼            ▼            ▼
                                  idle        working     auth_required
                                   │            │            │
                                   ▼            ▼            ▼
                              串口发"G"    串口发"Y"    串口发"R"
                                   │            │            │
                                   ▼            ▼            ▼
                              STM32 绿灯    STM32 黄灯    STM32 红灯
```

### ESP32 核心代码

```cpp
void update_agent_status() {
    HTTPClient http;
    http.begin("http://你的服务器:8000/agent/status");
    int code = http.GET();
    
    if (code == 200) {
        String json = http.getString();
        // 简易 JSON 解析（或引入 ArduinoJson 库）
        if (json.indexOf("\"status\":\"idle\"") > 0) {
            Serial2.print("$LED,GREEN\r\n");
        } else if (json.indexOf("\"status\":\"working\"") > 0) {
            Serial2.print("$LED,YELLOW\r\n");
        } else if (json.indexOf("\"status\":\"auth_required\"") > 0) {
            Serial2.print("$LED,RED\r\n");
        }
    }
    http.end();
}

void loop() {
    static unsigned long last = 0;
    if (millis() - last > 5000) {
        last = millis();
        update_agent_status();
    }
}
```

## OpenCode 集成方案

### 场景

机器人在 STM32 中运行，需要决策时通过 ESP32 → WiFi → 调用 OpenCode API → 获得指令 → 控制执行器。

### 架构

```
┌──────────────────────────────────────────────────┐
│  机器人 (STM32+ESP32)                              │
│  ├─ STM32: 传感器采集 → 打包数据 → UART→ESP32    │
│  └─ ESP32: 收数据 → HTTP POST /opencode/query     │
│            → 收到决策指令 → UART→STM32 → 执行     │
└──────────────────┬───────────────────────────────┘
                   │ WiFi
                   ▼
┌──────────────────────────────────────────────────┐
│  PC/服务器 运行 OpenCode Agent                     │
│  ├─ 收到机器人传来的传感器数据+当前任务            │
│  ├─ OpenCode 决策（调用 LLM）                     │
│  └─ 返回指令：移动/说话/亮灯                      │
└──────────────────────────────────────────────────┘
```

### 开放 API 接口

让 OpenCode 暴露一个 HTTP 接口：

```python
# opencode_agent_server.py
from flask import Flask, request, jsonify
import subprocess

app = Flask(__name__)

@app.route('/agent/status', methods=['GET'])
def agent_status():
    # 返回 Agent 当前状态
    return jsonify({"status": "idle"})

@app.route('/agent/decide', methods=['POST'])
def agent_decide():
    data = request.json
    # data: {"sensors": {"distance": 50, "angle": 90}, "task": "navigate"}
    # 调用 OpenCode CLI 做决策
    result = subprocess.run(
        ['opencode', 'decide', '--input', str(data)],
        capture_output=True, text=True
    )
    return jsonify({"action": result.stdout.strip()})

if __name__ == '__main__':
    app.run(host='0.0.0.0', port=8000)
```

> 实际方案取决于你使用的 OpenCode 版本和 API 接口，具体请查看 OpenCode 文档。

## 语音控制方案

### 方案 A：SU-03T 离线语音模块

- 不需要网络，本地识别预设命令词
- 通过串口输出识别结果
- 命令词需要提前烧录（厂家工具配置）

**流程**：说"开灯" → SU-03T 串口发 `0xAA 0x01` → STM32 解析 → 开灯

### 方案 B：ESP32 在线语音识别

- ESP32 通过 I2S 麦克风录音
- HTTP 上传音频到 Whisper API → 返回文本
- 然后调用 AI API 做语义理解和决策

**优点**：可以说任意话，不需要预设命令词
**缺点**：需要联网，有延迟
